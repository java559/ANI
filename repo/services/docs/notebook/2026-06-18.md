# 前端开发日志（2026.06.18）

> **代码落点**：`repo/frontends/console/`、`repo/frontends/boss/`  
> **规范依据**：[ANI-SERVICES-TEAM-GUIDE.md](../../../../ANI-SERVICES-TEAM-GUIDE.md)

### 今日目标

- 理清 ANI 项目前端入口与目录职责
- 按《ANI Services 团队开发指南》对齐 API 客户端

### 完成内容

- 删除误迁入的 `App.tsx` 与 `demo/` 演示壳，Console 统一使用 `src/routes/` + TanStack Router
- 接入 `@tanstack/router-plugin`，`main.tsx` 挂载 `RouterProvider`
- 建立 **Services API 客户端**：`api/client.ts`，`baseUrl: /api/v1/svc`，类型来自 `services/v1.yaml` → `schema.d.ts`
- 建立 **Core API 客户端**：`api/coreClient.ts`，`baseUrl: /api/v1`，类型来自 `v1.yaml` → `core-schema.d.ts`
- 新增 `api/auth.ts`，`setAuthToken` 同时作用于 Services 与 Core 请求
- 新增 `scripts/gen-core-schema.mjs`（生成 Core 类型时对已知 YAML 语法做规范化，不修改 `v1.yaml`）
- 示例页面接入 Core API：`/usage`、`/registry`、`/settings/api-keys`
- Makefile 增加 `gen-console-api`，同时生成两种前端类型
- 建立 `services/docs/notebook/` 目录，用于按日记录前端开发日志

### 遇到的问题 / 结论

- 仓库曾有两套前端入口（`App.tsx` Demo vs `routes/`），已统一为 `routes/`
- `v1.yaml` 中 `secondary_color:{ type` 缺少空格，导致 openapi-typescript 无法直接解析；通过生成脚本规范化处理
- 业务页用 `api`（`/api/v1/svc`），基础设施页用 `coreApi`（`/api/v1`）
